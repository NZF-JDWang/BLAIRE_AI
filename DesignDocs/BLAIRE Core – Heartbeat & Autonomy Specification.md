## Heartbeat Purpose

Simulate agency and proactive behavior.

Triggered by:
- Scheduler (cron or internal loop)

---

## Heartbeat Modes

1. Maintenance
2. Reflection
3. Project Work
4. Social

---

## Hard Constraints

- Quiet hours enforced
- Max notifications per day
- Tool rate limits
- No destructive actions in v1

---

## Heartbeat Decision Flow

1. Determine allowed modes
2. Choose mode (probability + priorities)
3. Execute safe actions
4. Log to episodic memory
5. Optionally notify user
